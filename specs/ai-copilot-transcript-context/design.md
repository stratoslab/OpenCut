# Design: AI Co-Pilot Transcript Context

## Overview

Wire the Whisper transcript text — already generated by the existing transcription pipeline — into the AI Co-Pilot's plan generation context. Three changes are needed: (1) store the transcript on the scene data model so it persists across sessions, (2) set it from the captions panel after transcription completes, and (3) read it in the AiAgentPanel when building the ProjectContext. The type system and prompt template are already prepared for this — only the data flow is missing.

## Architecture

```
Captions Panel (assets-view.tsx)
  │
  │ After transcription completes
  │
  ▼
Scene.transcript  ◄──── TScene interface (timeline/types.ts)
  │                   SerializedScene (storage/types.ts)
  │                   Auto-persisted via existing save/load cycle
  │
  ├──► AiAgentPanel (ai/components/AiAgentPanel.tsx)
  │       Reads scene.transcript.fullText → ProjectContext.transcript
  │       → agent.generatePlan() → buildPlanPrompt() includes transcript
  │
  └──► GemmaChatPanel (transcript-editor/GemmaChatPanel.tsx)
        Reads from same source (existing prop interface)
```

## Components

### Component 1: `TScene` — Add `transcript` field

- **File:** `apps/web-vite/src/timeline/types.ts`
- **Change:** Add `transcript?: WordTranscript` to the `TScene` interface
- **Responsibility:** Carries the scene's transcript as part of the scene model, automatically serialized/deserialized by the existing save/load cycle
- **Rationale:** The scene is the natural scope for a transcript — each scene has its own video content and its own captions track. Placing it here means zero additional persistence infrastructure.
- **Dependencies:** Imports `WordTranscript` from `@/transcription/types` — no circular dependency risk (transcription is a leaf module)

### Component 2: `Captions` panel — Store transcript on scene after transcription

- **File:** `apps/web-vite/src/subtitles/components/assets-view.tsx`
- **Change:** After `transcriptionService.transcribe()` succeeds, call `transcribeToWords()` (or build `WordTranscript` from the existing result) and set it on the active scene via `editor.scenes.updateScene()`
- **Responsibility:** Bridge between the transcription pipeline and the scene data model
- **Dependencies:** `transcriptionService.transcribeToWords()`, `editor.scenes.updateScene()`

**Design decision — two approaches considered:**

| Approach | Complexity | Data duplication | Persistence |
|----------|-----------|-----------------|-------------|
| **A: Store on scene** | Low — one field on TScene | None — single source of truth | Free — auto-saves with project |
| **B: Separate Zustand store** | Medium — parallel save/load | High — must sync with scene state | Manual — must wire into StorageService |

**Chosen: Approach A** — lowest complexity, least duplicated code, and the transcript naturally belongs to the scene concept.

### Component 3: `AiAgentPanel` — Read transcript and pass to context

- **File:** `apps/web-vite/src/ai/components/AiAgentPanel.tsx`
- **Change:** After reading `editor.scenes.getActiveScene()`, extract `scene.transcript?.fullText` and include it in the `context` object passed to `agent.generatePlan()`
- **Responsibility:** Complete the data flow from scene model → LLM prompt
- **Dependencies:** Already has `useEditor()` access to scenes

### Component 4: `StorageService` — Serialize transcript with scene

- **Files:** `apps/web-vite/src/services/storage/service.ts`, `apps/web-vite/src/services/storage/types.ts`
- **Change:** `SerializedScene` already uses `Omit<TScene, ...>` so adding `transcript?: WordTranscript` to `TScene` propagates to the serialized type automatically. However, the explicit map in `saveProject()` (lines 138-146) and `loadProject()` (lines 197-206) must be updated to pass through the `transcript` field
- **Responsibility:** Ensure transcript survives page refresh

## Data Models

### Changes to `TScene` (timeline/types.ts)

```typescript
import type { WordTranscript } from "@/transcription/types";

export interface TScene {
  id: string;
  name: string;
  isMain: boolean;
  tracks: SceneTracks;
  bookmarks: Bookmark[];
  transcript?: WordTranscript;  // NEW — optional, set after transcription
  createdAt: Date;
  updatedAt: Date;
}
```

### No changes to storage types

`SerializedScene = Omit<TScene, "createdAt" | "updatedAt"> & { ... }` — the `transcript` field is optional so it flows through existing type definitions without modification. If no transcript exists, it's omitted from the serialized blob.

## Data Flow

### Save cycle (auto-save after transcription)

```
Captions panel
  → transcriptionService.transcribe()  → TranscriptionResult
  → transcriptionService.transcribeToWords()  → WordTranscript
  → editor.scenes.updateScene(sceneId, { transcript: wordTranscript })
  → command-manager → markDirty()
  → save-manager (800ms debounce)
  → storage-service.saveProject()
    → SerializedProject { scenes: [{ ...tracks, transcript }] }
    → IndexedDB
```

### Load cycle (page refresh)

```
editor.project.loadProject()
  → storage-service.loadProject()
    → Deserialize: scene.transcript = serializedScene.transcript || undefined
  → SceneManager restores transcript on scene object
```

### AI plan generation

```
User types goal → "Cut out the intro"
  → AiAgentPanel.handleGeneratePlan()
    → const scene = editor.scenes.getActiveScene()
    → const context = {
        duration, trackCount, mainTrackElements, audioTrackCount,
        transcript: scene.transcript?.fullText  // NEW
      }
    → agent.generatePlan(goal, context)
      → buildPlanPrompt(goal, context)
        → `- Transcript: ${transcript ?? "none"}`  // already exists!
```

## Key Algorithms

### Transcript length management

Long transcripts (podcasts, lectures, interviews) could exceed the LLM's context window. Strategy:

1. **Token estimation:** Estimate transcript token count as `fullText.length / 4` (rough approximation for English)
2. **Threshold check:** If estimated tokens > 3,000 (leaving room for system prompt + user goal), truncate
3. **Truncation strategy:** Head-only (keep the first N characters). For a video editor AI, the beginning of the transcript is usually most relevant (intros, setup). If needed, this could later become a "head + tail" approach.
4. **Edge case:** If even the head-only transcript with the system prompt exceeds the model's max tokens, exclude it entirely and fall back to `undefined` (current behavior)

Implementation:

```typescript
function getTranscriptContext(scene: TScene): string | undefined {
  if (!scene.transcript?.fullText) return undefined;
  const MAX_CHARS = 12_000;  // ~3,000 tokens
  if (scene.transcript.fullText.length <= MAX_CHARS) {
    return scene.transcript.fullText;
  }
  return scene.transcript.fullText.slice(0, MAX_CHARS) + "... [truncated]";
}
```

### WordTranscript construction from existing transcription

The captions panel currently calls `transcriptionService.transcribe()` which returns `TranscriptionResult` (segments without word-level timing). To build a `WordTranscript`, call `transcriptionService.transcribeToWords()` instead — this method already exists and internally calls `segmentsToWordSegments()` + `buildWordTranscript()`.

## Error Handling

| Situation | Handling |
|-----------|----------|
| No transcript exists | `context.transcript` = `undefined` → prompt renders "none" (existing behavior) |
| Transcription cancelled mid-way | No `WordTranscript` stored on scene — Co-Pilot sees `undefined` |
| Scene switched during plan generation | Snapshot already taken at `handleGeneratePlan` time — stale but safe |
| Transcript contains special JSON chars | `buildPlanPrompt()` already interpolates via template literal — no injection risk since it's in a code block within a string. But `fullText` could contain backticks or `${...}` that break the template literal. **Fix:** escape backticks and `${}` in the transcript text before interpolation |
| Very long transcript | Truncated via heuristic above |
| Persistent storage migration | Since `transcript` is optional on `TScene`, old projects without it load correctly. No migration needed |

## Testing Strategy

- **Unit test:** `AiAgentPanel` context construction includes transcript when scene has one, excludes when it doesn't
- **Unit test:** `buildPlanPrompt()` output is valid JSON-compatible regardless of transcript content (especially edge characters like backticks, `${}`, quotes)
- **Unit test:** Long transcript truncation produces correct head-only result with `[truncated]` suffix
- **Integration test:** Full flow — captions generate → transcript stored → Co-Pilot plan includes transcript context in prompt
- **Property-based test:** For any `WordTranscript` (including empty, special chars, very long), the stored `transcript` field round-trips through serialize/deserialize without data loss
