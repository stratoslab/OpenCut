# Design: Smart Suggestions

## Overview

A suggestion queue system with a floating card component. Suggestions are generated by analyzing the project state (transcript, timeline, audio) and queued by severity. The card component shows one suggestion at a time with auto-dismiss timer.

## Components

### Component 1: SuggestionQueue
- **Responsibility:** Store, sort, and serve suggestions by severity
- **Interface:** `add(suggestion)`, `next(): Suggestion | null`, `dismiss()`
- **Key design:** Priority queue sorted by severity (warning=0, improvement=1, info=2)

### Component 2: SuggestionCard
- **Responsibility:** Display single suggestion with severity icon, description, Apply/Dismiss buttons, auto-dismiss timer
- **Interface:** React component with hover-aware timer

### Component 3: SuggestionGenerators
- **Responsibility:** Analyze project state and generate suggestions
- **Examples:** "Long silence detected at 2:30", "Filler words found in first 30 seconds", "Audio levels inconsistent"

## Data Model

```typescript
interface Suggestion {
  id: string;
  severity: "warning" | "improvement" | "info";
  title: string;
  description: string;
  apply: () => Promise<void>;
}
```

## Data Flow

1. Project state changes → suggestion generators analyze for issues (silence, filler words, audio levels)
2. Results queued by severity → SuggestionCard displays highest-priority item
3. User clicks Apply → execute action → next suggestion shown
4. User clicks Dismiss or timeout fires → next suggestion shown

## Error Handling

| Situation | Handling |
|-----------|----------|
| No suggestions generated | Keep queue empty — no card shown |
| Apply action fails | Log to diagnostics panel, dismiss that suggestion, show next |
| All suggestions dismissed/timeout | Hide card until new suggestions generated |

## Testing Strategy

- **Unit test:** SuggestionQueue returns items in severity order (warning > improvement > info)
- **Unit test:** Auto-dismiss timer fires at 30s, pauses on hover, resumes on mouse leave
- **Property-based test:** For any queue of N suggestions with random severities, `next()` SHALL always return the highest remaining severity
- **Integration test:** Generator produces suggestion → card appears → Apply modifies timeline → undo works
